Buttons

Action elements using DM Sans font with hover underline states for clear interaction feedback.

What Are Buttons?

Buttons trigger actions when clicked. They guide users through workflows like placing orders, requesting quotes, and confirming transactions. All buttons use DM Sans font and include hover underline states for clear visual feedback.

Primary Buttons

Main call-to-action buttons with Waves-500 (#025E96) background. Hover shows Waves-700 (#01385A) with underline.

HTML 
<!-- Small (32px) -->
<button class="btn btn-primary btn-sm">Small</button>

<!-- Medium (40px) - Default -->
<button class="btn btn-primary btn-md">Medium</button>

<!-- Large (48px) -->
<button class="btn btn-primary btn-lg">Large</button>

<!-- Disabled -->
<button class="btn btn-primary btn-md" disabled>Disabled</button>
React + JSX 
const PrimaryButton = ({ size = 'md', children, disabled, onClick }) => (
  <button
    className={`btn btn-primary btn-${size}`}
    disabled={disabled}
    onClick={onClick}
  >
    {children}
  </button>
);

// Usage
<PrimaryButton size="sm">Small</PrimaryButton>
<PrimaryButton>Medium (Default)</PrimaryButton>
<PrimaryButton size="lg">Large</PrimaryButton>
<PrimaryButton disabled>Disabled</PrimaryButton>

CSS Variables Used

Property Value
font-family var(--font-heading) → DM Sans
background var(--color-waves-500) → #025E96
background (hover) var(--color-waves-700) → #01385A
text-decoration (hover) underline with 4px offset

Light Buttons (For Dark Backgrounds)

Use on dark backgrounds like Waves-500 (#025E96). Available in fill and outline variants.

HTML 
<!-- On dark background -->
<div style="background: #025E96; padding: 32px;">
  <button class="btn btn-light-fill btn-md">Light Fill</button>
  <button class="btn btn-light-outline btn-md">Light Outline</button>
</div>
React + JSX 
const LightButton = ({ variant = 'fill', size = 'md', children, disabled, onClick }) => (
  <button
    className={`btn btn-light-${variant} btn-${size}`}
    disabled={disabled}
    onClick={onClick}
  >
    {children}
  </button>
);

// Usage
<LightButton variant="fill">Light Fill</LightButton>
<LightButton variant="outline">Light Outline</LightButton>

Tertiary Buttons (Stroke Outline)

Secondary actions with stroke outline. Use anywhere for less prominent actions.

HTML 
<button class="btn btn-tertiary btn-md">Tertiary Button</button>

Best Practices

✅ Do

  • Use clear, action-oriented labels ("Add to Cart", "Request Quote")
  • Keep labels concise (2-4 words maximum)
  • Use only ONE primary button per section
  • The hover underline provides clear feedback
  • Ensure 44x44px minimum touch targets
  • Use sentence case ("Add to cart")

❌ Don't

  • Don't use vague labels ("Click here", "OK")
  • Don't make buttons too small (32px minimum)
  • Don't use multiple primary buttons
  • Don't use all caps text
  • Don't disable without explanation
  • Don't place buttons too close (8px minimum gap)
