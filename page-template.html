Best Practices

✅ Do

  • Use clear, action-oriented labels ("Add to Cart", "Submit Order")
  • Keep labels concise (2-4 words maximum)
  • Maintain consistent sizing throughout your interface
  • Provide visual feedback on hover and click
  • Ensure adequate spacing between interactive elements (minimum 8px)

❌ Don't

  • Don't use vague labels like "Click here" or "Submit"
  • Don't make buttons too small (minimum 32px height for touch)
  • Don't use multiple primary buttons on one screen
  • Don't use all caps for button text (reduces readability)
  • Don't disable buttons without explanation

♿ Accessibility Guidelines

  • ✅ Minimum size: 44x44px touch target (WCAG 2.5.5)
  • ✅ Color contrast: 4.5:1 minimum for text, AAA compliant
  • ✅ Keyboard navigation: Focusable via Tab, activatable with Enter/Space
  • ✅ Screen readers: Use descriptive aria-labels when needed
  • ✅ Focus indicator: Visible 3px outline on focus
  • ✅ States: Clear visual feedback for hover, active, disabled states
